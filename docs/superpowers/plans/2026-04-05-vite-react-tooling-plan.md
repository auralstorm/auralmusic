# Vite React Tooling Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Initialize a `pnpm`-managed `Vite + React + TypeScript` project with linting, formatting, Git hooks, commit message enforcement, and `release-it`-based changelog releases.

**Architecture:** Start from a single Vite application, then layer tooling in isolated config files so formatting, linting, commit validation, and release automation stay decoupled. Because this phase does not include a test framework, verification uses command-based red/green checks for lint, build, Git hooks, commitlint, and release dry-runs instead of unit tests.

**Tech Stack:** Vite, React, TypeScript, pnpm, ESLint flat config, Prettier, Husky, lint-staged, commitlint, Commitizen (`cz`), release-it

---

## File Map

- `package.json`: scripts, dependencies, Commitizen entry, release command
- `pnpm-lock.yaml`: lockfile generated by dependency installation
- `.gitignore`: ignore build output, dependencies, env files, and local brainstorm artifacts
- `vite.config.ts`: Vite configuration
- `tsconfig.json`: root TypeScript project references
- `tsconfig.app.json`: browser app compiler settings
- `tsconfig.node.json`: node-side compiler settings for tooling files
- `eslint.config.js`: ESLint flat config for TypeScript and React
- `.prettierrc.json`: formatting rules
- `.prettierignore`: files and folders excluded from Prettier
- `.lintstagedrc.json`: staged-file lint/format routing
- `commitlint.config.js`: Conventional Commits validation rules
- `.czrc`: Commitizen adapter configuration
- `.release-it.json`: release automation and changelog generation config
- `.husky/pre-commit`: staged-file validation hook
- `.husky/commit-msg`: commit message validation hook
- `CHANGELOG.md`: release history file managed by `release-it`
- `README.md`: setup, commit, and release usage instructions
- `src/main.tsx`: app entry
- `src/app/App.tsx`: top-level application component
- `src/styles/index.css`: global styles
- `src/components/.gitkeep`: preserve shared-component directory
- `src/features/.gitkeep`: preserve feature-module directory
- `src/lib/.gitkeep`: preserve shared-library directory
- `src/pages/.gitkeep`: preserve route-container directory
- `src/assets/.gitkeep`: preserve asset directory

## Scripts

- `pnpm dev`: start the development server
